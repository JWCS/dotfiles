# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository Purpose

Personal dotfiles for rapid deployment across machines (Ubuntu/Debian, WSL2, remote servers). Managed via shell scripts — no build system or test suite.

## Setup Scripts

- `new_machine_setup.sh` — Bootstrap bare system (curl, git, basic tools); run first on a fresh machine
- `setup.sh` — Links configs and initializes tool configs; safe to re-run
- `install.sh` — Installs individual tools; can run selectively:
  ```sh
  ./install.sh git tmux vim
  # or
  source install.sh && install-nvim
  ```
- `setup_root.sh` — Mirrors user dotfiles to /root via symlinks

## Architecture

### Deployment Strategy

- Tool configs in subdirectories: `git/`, `docker/`, `tmux/`, `bash/`
- `setup.sh` symlinks configs (tmux, vim, inputrc) or sources them (bash)
- `~/.bashrc` is generated by `setup.sh` to source files from this repo — not symlinked
- `~/.gitconfig` is generated with `[include]` directives pointing into `git/`

### Feature Flag System

Optional features are enabled via environment variables set before sourcing:

**Vim (`.vimrc.plug`):**
- `VRC_FEAT_COPILOT` — GitHub Copilot plugin
- `VRC_FEAT_COC` — Coc.nvim LSP
- `VRC_FEAT_CMAKE` — CMake tools
- `VRC_LANG_SH` — Shell syntax/checking

**Bash (`.bashrc_features`):**
- `BRC_FEAT_KEYCHAINS` — Keychain SSH agent management
- `BRC_FEAT_GIT_PROXY` — SOCKS5 git proxy
- `BRC_FEAT_WSL` — WSL2-specific features

### Key Config Files

| File | Purpose |
|------|---------|
| `.bashrc_common` | PS1, history (eternal, timestamped), core aliases |
| `.bashrc_features` | Tool integrations (pyenv, nvm, direnv, glow) |

| `bash/util.sh` | Helpers: `_has_cmd`, `addToPATH`, `sed-all`, `alias_nice` |

### Vim Plugin Manager

vim-plug auto-installs on first launch. Plugins are declared in `.vimrc.plug`. Core always-installed plugins: vim-sensible, vim-obsession, vim-repeat, auto-pairs, vim-lastplace.

### Tmux Plugin Manager

TPM auto-initializes via `setup.sh`. Plugins: tmux-sensible, tmux-resurrect, tmux-continuum.

## Neovim Config (`nvim/`)

Config lives in `nvim/` and is symlinked to `~/.config/nvim` by `setup.sh` (`_nvimconf_init`).

### Structure

```
nvim/init.lua              # bootstraps lazy.nvim, loads opts/keys/autocmds
nvim/lua/opts.lua          # settings (translated from .vimrc.min)
nvim/lua/keys.lua          # keymaps (space leader, jj ESC, C-hjkl, telescope, diagnostics)
nvim/lua/autocmds.lua      # filetype settings, strip-whitespace, VRC_LANG_SH flag
nvim/lua/plugins/
  core.lua                 # treesitter, gitsigns, vim-tmux-navigator, obsession, snacks
  telescope.lua            # fuzzy find (bottom_pane layout)
  lsp.lua                  # mason + mason-lspconfig + nvim-cmp
  dap.lua                  # nvim-dap + dap-ui + virtual-text (lazy, key-triggered)
```

### Key Notes

- Uses **nvim 0.11 native API**: `vim.lsp.config` + `vim.lsp.enable` (not deprecated lspconfig framework)
- Same `VRC_FEAT_*` env var flags work in Lua via `vim.env.*`
- `treesitter` loads on `BufReadPre`/`BufNewFile`; parsers auto-install on first file open
- `nvim-dap` loads only when a DAP keymap is first pressed (no startup overhead)
- `vim-tmux-navigator` requires `@plugin 'christoomey/vim-tmux-navigator'` in tmux (already added)
- LSP servers installed via mason: clangd, pyright, bashls, lua_ls, yamlls, jsonls, taplo
  - cmake LSP: uncomment in `lsp.lua` after `pyenv install 3.x`

## Owner Preferences

This section encodes working style for use across all machines and projects.

### Development Environment

The IDE is **tmux + nvim + gdb + CLI tools**. Do not suggest:
- GUI applications, VS Code, JetBrains, or GUI debuggers
- However, a very nice gui or debugger that's complementary not redundant may be suggested.
- File tree plugins (neotree, nvim-tree) — use telescope or netrw
- Auto-opening panels or sidebars — UI is opened manually
- Notification popups or status overlays beyond minimal indicators

Target platform is always **Ubuntu/Debian**, and primarily remote headless SSH. Native linux device or wsl2 also used. Uses xclip/tmux for clipboarding. Install commands default to apt/deb/binary; avoid snaps.

### Vim/Nvim

- **Muscle memory is preserved**: space leader, `jj` ESC, `C-hjkl` windows, `<leader><cr>` clear search
- When adding a nvim plugin that replaces a vim plugin, prefer the nvim-native equivalent
- New optional features go under a `VRC_FEAT_*` env var flag — disabled by default
- Lazy-load plugins that aren't needed at startup (keymaps, events, not `lazy=false`)
- Completion: always provide both smart (LSP) and dumb (buffer words) sources together

### Code Style

- Minimal and focused — only change what's needed, no speculative abstractions
- Document *why* in comments, not *what* (the code shows what)
- New tools and workflows documented in-config where they're used, not in separate files
- Feature flags over hardcoded assumptions — same config must work on minimal and full machines

### Languages

Primary: bash, C, C++, Python, CMake, Ninja, Makefile, Dockerfile
Data/config: YAML, TOML, JSON, Markdown, gitconfig

### General

- Willing to learn and adopt new tools — but document them, don't just add them silently
- Prefer scriptable and reproducible over convenient and manual
- When in doubt, do less and ask — don't make structural changes without confirmation
